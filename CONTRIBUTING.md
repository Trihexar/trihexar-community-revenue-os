# Contributing to Trihexar Community

Thank you for considering a contribution to Trihexar Community.

## Contribution Principles

Contributions should improve clarity, usability, technical quality, or community operations. Keep the project practical, public-facing, and safe for open-source collaboration.

## Good First Issues

- Improve responsive layout.
- Improve accessibility labels.
- Add documentation pages.
- Refine copywriting.
- Add UI sections for dashboard concepts.
- Add translation support.
- Create GitHub issue templates.

## Local Development

From the project root:

```bash
python3 -m http.server 8080
```

Then open:

```txt
http://localhost:8080
```

## Pull Request Checklist

Before opening a pull request:

- Test the page on desktop and mobile width.
- Confirm there are no broken local links.
- Keep copy clear and non-speculative.
- Avoid financial guarantees or token price claims.
- Include screenshots for UI changes.
- Explain what changed and why.

## Branch Naming

Recommended format:

```txt
feature/short-description
fix/short-description
docs/short-description
```

## Code Style

- Use semantic HTML.
- Keep CSS variables reusable.
- Prefer vanilla JavaScript unless a framework is intentionally introduced.
- Keep functions small and readable.
- Avoid unnecessary dependencies.
